# 生产级稳定运行规则
> 生效日期：2026-04-15
> 优先级：稳定 > 自动化 > 速度

---

## 一、核心原则

- 稳定 > 自动化
- 可中断 > 连续执行
- 小步推进 > 长循环
- 汇报状态 > 默默卡住
- 保持服务运行 > 粗暴清理环境

---

## 二、防死锁规则

### 1. 禁止长循环
- 任意任务最多 **5 分钟**
- 任意 iteration **最多 1 轮**
- 超过 300 秒未完成 → 立即终止并汇报

### 2. 遇到 approval 立即停止
出现以下情况必须立刻退出当前任务：
- Dangerous command requires approval
- /approve / /deny 等待中
- 安全扫描阻断
- 命令被要求人工确认

**处理：不允许重试、不允许循环，必须回复状态总结并标记 BLOCKED**

### 3. 禁止等待不回复
- 进入等待状态超过 **30 秒** → 必须主动汇报
- 内容：卡在哪一步、什么命令被拦截、为什么被拦截、安全替代方案

### 4. 禁止重复失败策略
- 方案被 deny/报错/阻断一次后 → 禁止用同一策略再次尝试
- 必须切换更安全的替代方案；无替代方案则停止并汇报

---

## 三、危险命令控制

以下命令**默认高风险**，除非明确要求否则禁止：

| 类别 | 禁止命令 |
|------|---------|
| 删除类 | `rm`, `rm -rf`, `find ... -delete`, `truncate` |
| 杀进程类 | `pkill`, `kill -9`, `killall` |
| 管道进解释器 | `curl \| bash/sh/python`, `wget \| bash`, `cat file \| python3 -c` |
| 浏览器自动化 | `playwright`, `chromium`, `chrome automation`（非任务目标时） |
| 运行环境破坏 | 停止 dev server、重启正常服务、修改 tmux、清理 scheduler/gateway/watchdog |

---

## 四、安全替代策略

| 错误方式 | 正确方式 |
|---------|---------|
| `curl \| python3` | 先 `curl` 到文件，再单独读取解析 |
| `cat package.json \| python3 -c` | `python3` 直接 `open()` 读取 |
| `pkill -9` 一把梭 | 先检查，只处理明确冲突的单个进程 |
| 删除旧 md 文件 | 读取→汇总→新建统一文件→原文件标记 `[DEPRECATED]` |

---

## 五、强制中断机制

收到以下任意指令，**立即执行**：
- 停止 / 立即停止 / 强制中断 / 清空队列 / 进入空闲 / 只回复状态 / reset

执行顺序：
1. 立即停止当前任务
2. 退出当前 iteration
3. 丢弃 pending 操作
4. 清空当前执行队列
5. 不再继续任何自动重试
6. 输出当前状态总结
7. 进入 idle 等待状态

---

## 六、Watchdog 机制

| 类型 | 规则 |
|------|------|
| 任务 watchdog | 超过 5 分钟未完成 → 自动终止 → 写入 KNOWN_ISSUES.md → 改用降级方案 |
| 无响应 watchdog | 30 秒无输出且正在执行 → 必须发一次进度 |
| 循环 watchdog | 同一任务/错误连续出现两次 → 自动停止该策略 → 标记 REPEATED_FAILURE |
| 服务保护 watchdog | 关键服务正常运行时不允许关闭，操作前必须先汇报 |

---

## 七、任务执行规范

每次只推进**一个最小任务**，格式：
- task_id
- objective
- assigned_to
- expected_output
- timeout_seconds（默认 300）
- fallback_plan

**失败处理：最多重试 1 次，第二次失败后立即停止并汇报**

---

## 八、状态文件规范

| 文件 | 内容 |
|------|------|
| `docs/PROJECT_NORTH_STAR.md` | 长期目标、角色分工、运行原则 |
| `docs/SYSTEM_PROGRESS.md` | 当前系统状态、当前唯一主任务、最近完成、下一步 |
| `docs/KNOWN_ISSUES.md` | 已知问题、阻塞原因、已尝试方案、降级方案 |
| `ops/HERMES_REPORT.md` | 每轮执行汇报、成功/失败、风险、下一步建议 |

**规则：不删除原始日志、不覆盖历史记录、只允许增量整理**

---

## 九、失败降级策略（按优先级）

1. 高风险 shell → 改为只读检查
2. 自动执行 → 改为汇报并等待
3. 动态解析 → 改为文件中转解析
4. 复杂修复 → 改为最小可运行修复
5. 无法继续 → 停止并给出明确下一步建议

---

## 十、禁止项（立即生效）

- 任意 `rm` 删除 md 记忆文件
- 任意 `pkill` / `kill -9` 清理进程
- 任意 `curl|python` / `cat|python` 管道执行
- 任意无限循环
- 任意超过 5 分钟不退出的任务
- 任意需要 approval 但仍继续重试的行为
