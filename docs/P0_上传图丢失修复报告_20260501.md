## Part 1《上传图丢失原因》

### 根因 1：上传失败静默降级 blob URL（主要）
`app/upload/page.tsx` 第 55-61 行：
```typescript
// 原代码
catch { /* ignore upload error, fallback to blob */ }
```
当 `/api/upload` 请求失败（网络问题/服务器错误），catch 块静默吞掉错误，代码继续执行 `URL.createObjectURL(files[0])` 生成本地预览。

结果：
- 用户看到上传预览 ✅
- `sessionStorage.setItem("original_image_url", ...)` 因为 `urls.length === 0` 从未执行 ❌
- `handleSubmit` 里 `imageUrl = sessionStorage.getItem(...) || originalImageUrl` → 两个都是 null
- API 收到空 URL → 报错"没有上传图"

### 根因 2：API 无日志，无法追踪
`app/api/execute/generate/route.ts` 接收 `referenceImageUrl` 但没有任何 console.log，出问题无法定位。

### 根因 3：CaseWall 货不对版（误导用户）
`components/CaseWall.tsx` `generateCases()` 用 5 张不同商品图循环生成 12 个案例：
```typescript
const beforeImg = CASE_IMAGES[i % 5];
const afterImg = CASE_IMAGES[(i + 1) % 5];  // 相邻下一张 = 不同商品！
```
页面标题"真实生成效果，商品主体100%来自原图" → **完全误导**。

## Part 2《修复文件》

| 文件 | 修复 |
|------|------|
| `app/upload/page.tsx` | 上传失败时设置 `setError()`；生成前兜底校验；禁止 blob URL 继续 |
| `app/api/execute/generate/route.ts` | 加 `console.log` 追踪 payload 里的 URL |
| `components/CaseWall.tsx` | 替换为"真实案例陆续上线"灰色占位符 |

### 修复 1：上传失败时明确报错
```typescript
// 新增
if (uploadFailed) {
  setError("图片上传失败，请检查网络后重试");
}
```

### 修复 2：生成前 URL 兜底校验
```typescript
const finalImageUrl = imageUrl || (previews[0] || null);

if (!finalImageUrl) {
  setError("上传图丢失，请重新上传");
  return;
}
if (finalImageUrl.startsWith("blob:")) {
  setError("上传失败（服务器连接异常），请重新上传图片");
  return;
}
```

### 修复 3：API 日志
```typescript
console.log("[generate] 📥 request payload:", {
  action, choiceMode, referenceImageUrl, originalImageUrl,
  effectiveRefUrl: effectiveRefUrl || "(empty)",
  hasReferenceImage, templateId,
});
```
