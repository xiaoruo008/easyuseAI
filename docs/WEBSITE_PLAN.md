### 页面2：诊断 `/diagnosis`

**作用：** 用5道业务题捕获用户痛点，判断转化方向

#### 题目内容（5道题）

**Q1：你现在最头疼的事情是什么？**
- A：写了没人看（流量问题）
- B：客户来问同一个问题（客服问题）
- C：每天手动整理数据（效率问题）
- D：想引流获客，但没有好内容（内容+流量）

**Q2：你每个月花多少时间在这件事上？**
- A：不到1小时（紧迫感低）
- B：1~5小时（有点烦）
- C：5小时以上（急需解决）
- D：每天都在处理（快崩溃了）

**Q3：你试过什么方法解决这个问题？**
- A：没试过，不知道从哪里下手
- B：试过一些方法，但效果不好
- C：正在用某个工具，但效率还是很低
- D：花钱请过人，但成本太高

**Q4：如果这个问题解决了，你最希望看到什么效果？**
- A：省时间
- B：多赚钱
- C：两者都要
- D：不焦虑

**Q5：你的预算范围是？**
- A：不想花钱，先了解免费方案
- B：500元以内，先试试效果
- C：500~2000元，效果好可以持续用
- D：2000元以上，我要最好的方案

#### 诊断结果类型（4种）

| 类型 | 判断条件 | 用户画像 |
|------|---------|---------|
| traffic | 流量题得分最高 | 有产品、有客户基础，但内容引流效率低的商家 |
| customer | 客服题得分最高 | 客户量大、咨询频繁，客服响应成为瓶颈的商家 |
| efficiency | 效率题得分最高 | 重复性工作繁重，渴望自动化提效的商家 |
| unclear | 得分接近 | 方向不明确，需要1对1梳理的商家 |

#### 按钮清单（诊断页）

| 按钮 | 位置 | 行为 |
|------|------|------|
| 返回（←箭头） | 顶部左侧 | 跳转 / |
| 4个选项按钮 | 问题区 | 选择后自动保存并跳转下一题 |

### 页面4：执行页 `/execute`

**作用：** 用户选择动作后，在这里生成图片或文案

#### 功能模块

**模块A — 头部信息**
- 返回按钮 → /result
- 显示当前动作名称和图标

**模块B — 用户情况摘要**
- 深色背景卡片
- 显示painPoint（核心问题）

**模块C — 免费次数用完提示**
- 当trial_count >= 2时显示
- 两个选项：
  - 方式1：留下联系方式（推荐），跳转/submit
  - 方式2：立即付费解锁 ¥99，跳转/result

**模块D — 图片任务输入（Image Task）**
- prompt输入框（textarea）
  - placeholder根据action变化：
    - product_photo：例如：放在纯白背景里，光线柔和，专业电商风
    - background_swap：例如：把背景换成现代简约的咖啡馆场景
- 风格选择：商业/极简/高端（3选1按钮，默认商业）
- 比例选择：1:1 / 3:4 / 16:9（3选1按钮，默认1:1）
- 产品原图上传（目前v1暂不支持，显示提示）

**模块E — 文案任务上传素材（Text Task）**
- 文件上传（可选）
- 支持格式：图片/PDF/Word/Excel

**模块F — 主按钮**
- 未出结果：立即帮我做一版（跳转/execute，调用API）
- 已出结果：再做一版

**模块G — 图片结果展示**
- 图片显示（1:1比例容器）
- 来源标注：mock（示例效果）/ 真实AI生成
- 操作按钮：复制链接 / 下载

**模块H — 文案结果展示**
- 标题 + 内容列表（每行带序号）

**模块I — 出结果后的底部按钮**
- 返回结果页
- 获取完整服务（跳转/submit）

## 四、商业模式与转化漏斗

### 用户旅程

```
1. 访问首页 → 看到效果对比图
       ↓
2. 点击"限量0元领取" → 进入诊断
       ↓
3. 完成5道题 → 获得诊断结果（traffic/customer/efficiency/unclear）
       ↓
4. 结果页点击某个动作（如"出商品图"）→ 进入执行页
       ↓
5. 执行页点击"立即帮我做一版" → 生成图片（mock结果，消耗1次免费试用）
       ↓
6. 免费次数用完（2次后）→ 出现留联系方式或付费的选项
       ↓
7a. 点击"留下联系方式" → /submit → 创建Lead
       或
7b. 点击"标准制作 ¥99" → 支付弹窗
       或
7c. 点击"完整交付 ¥299" → /result/pro → 支付弹窗
       ↓
8. 顾问在后台看到线索 → 联系用户 → 成交
```

## 五、核心文案策略

### 首页文案基调
- 不讲AI技术，讲"效果"
- 强调"0门槛"：0提示词、随手拍
- 限时优惠感：限量0元领取、前100名

### 诊断题基调
- 纯业务语言，不出现AI/技术/自动化等词
- 用户用直觉就能回答
- 不让用户觉得自己"不会用工具"

### 结果页文案基调
- 强痛点描述（让用户觉得被理解）
- 工作流具体化（不是功能罗列，是能解决什么问题）
- 示例内容可立即使用（不是demo，是真实可用的）
- 留联系方式是"获取更多"而不是"被迫注册"

### 执行页文案基调
- 制作中是"帮你做"而不是"AI在生成"
- 错误提示给用户台阶下（"网络波动"而非"系统错误"）
- 免费次数用完给用户两个选择，而不是堵死路
