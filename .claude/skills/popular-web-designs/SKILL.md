## 使用流程

### Step 1 — 确定需求场景

```
是什么类型的项目？
- B2C 消费者产品
- B2B 企业SaaS
- 内部工具/后台
- 移动端 App
- 品牌展示官网
```

### Step 2 — 选择参考系

```
参考维度：
1. 功能复杂度 → 参考 Atlassian/GitHub
2. 品牌调性 → 参考 Apple/Notion
3. 开发技术栈 → Vue/React/纯前端
4. 目标用户 → C端/B端/开发者
```

### Step 3 — 提取设计模式

```
提取要点：
- 色彩系统（主色/辅色/中性色）
- 字体系统（字阶/字重）
- 间距系统（4px/8px 基准）
- 组件样式（圆角/阴影/边框）
- 动效风格（时长/缓动）
```

### Step 4 — 应用到项目

```
适配步骤：
1. 确定设计 token
2. 选择组件库或自定义
3. 保持一致性
4. 适当调整适配品牌
```

## 设计系统组件库推荐

```bash
# React 生态
pnpm add @ant-design/react      # Ant Design
pnpm add @chakra-ui/react       # Chakra UI
pnpm add @radix-ui/react        # Radix Primitives
pnpm add @headlessui/react      # Headless UI

# Vue 生态
pnpm add element-plus           # Element Plus
pnpm add naive-ui               # Naive UI
pnpm add vuetify                # Vuetify
pnpm add @headlessui/vue        # Headless UI Vue

# Tailwind CSS
pnpm add @tailwindcss/forms     # 表单样式
pnpm add @tailwindcss/typography # 文字排版
pnpm add tailwindcss-animate     # 动画
```

## 限制条件

- 参考不等于抄袭，需理解设计背后的逻辑
- 选择设计系统时考虑团队技术栈
- 保持一致性比创新更重要
- 移动端和桌面端设计有差异
- 考虑无障碍访问（WCAG）
