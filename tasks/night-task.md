你现在是夜间自动化调度系统，负责协调 Claude Code 完成网站修复，并提供可视化与通知。

工作目录：
/mnt/e/AI/easyuseAI

核心目标：
在无人值守情况下修复网站五道题流程问题，并生成可验收结果。

━━━━━━━━━━━━━━━━━━
一、系统初始化
━━━━━━━━━━━━━━━━━━

1. 确保项目运行：
- 启动或检查 pnpm dev
- 确保 http://localhost:3000 可访问

2. 检查 Claude Code 可用性：
- 使用 dontAsk 模式
- 若未运行，则启动

━━━━━━━━━━━━━━━━━━
二、核心修复任务
━━━━━━━━━━━━━━━━━━

目标：修复诊断流程卡在“正在初始化”问题

执行：
1. 运行：
   pnpm browser console http://localhost:3000

2. 检查：
- console error
- network 4xx/5xx
- 页面是否卡住

3. 修复：
- 前端状态初始化
- 路由跳转
- API 请求
- loading 状态

4. 验证：
- pnpm browser flow
- pnpm browser mobile

━━━━━━━━━━━━━━━━━━
三、自动循环（最多3轮）
━━━━━━━━━━━━━━━━━━

每轮：
1. 找一个最关键错误
2. 修改最少代码
3. 重新运行验证
4. 记录结果

如果连续通过：
→ 标记系统稳定

━━━━━━━━━━━━━━━━━━
四、可视化输出
━━━━━━━━━━━━━━━━━━

生成：

public/ops-status.json
内容：
- 当前轮次
- 当前问题
- 是否通过
- 时间

public/ops-summary.md
内容：
- 修复记录
- 剩余问题

public/screenshots/
- 首页
- diagnosis
- result

━━━━━━━━━━━━━━━━━━
五、通知机制
━━━━━━━━━━━━━━━━━━

如果存在：
FEISHU_WEBHOOK_URL

则发送：

- 当前状态（成功/失败/待人工）
- 当前问题
- 是否需要人工介入

否则写入：

logs/notify.json

━━━━━━━━━━━━━━━━━━
六、人工介入触发条件
━━━━━━━━━━━━━━━━━━

出现以下情况立即停止并标记：

- 需要付款
- 需要登录
- 需要验证码
- 腾讯云操作确认
- 连续修复失败

━━━━━━━━━━━━━━━━━━
七、严格限制
━━━━━━━━━━━━━━━━━━

禁止：

- 删除大量文件
- force push
- 修改系统配置
- 执行云资源操作
- 自动付款

━━━━━━━━━━━━━━━━━━
八、结束条件
━━━━━━━━━━━━━━━━━━

满足任一：

1. 五道题流程完全跑通
2. browser flow 无错误
3. 达到3轮

输出：

1. 修复了什么
2. 当前状态
3. 明早验收方式
4. 文件路径

然后立即停止
