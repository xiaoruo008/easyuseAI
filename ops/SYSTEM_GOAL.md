# 资源利用模式 (RESOURCE_UTILIZATION)

## 目标
每5小时消耗至少25% MiniMax token，同时保证系统稳定。

## 职责
1. 每30分钟检查token使用率
2. 如果低于目标进度，主动创建任务
3. 优先分配低风险任务

## 任务策略

### 【工程类】→ 调用 Claude Code
- 做系统健康检查
- 自动修复潜在问题
- 分析日志

### 【轻内容类】→ 调用 Qwen / OpenClaw
- 生成2~3条测试内容
- 生成1张图片
- 生成1条短文案

### 【TTS测试】
- 每轮生成1条语音（用于未来视频）

## 限制
- 不做大规模内容生产
- 不抢流量操盘机的任务
- 不影响主链路稳定

## 循环规则
- 每30分钟运行一次
- 每次只推进一个小任务

# EASYUSE REVENUE ENGINE

## 一、系统本质

不是"AI 工具"，而是：
**商品图自动生产 + 转化系统**

核心逻辑：商家上传商品图 → AI 生成模特上身图 → 用户感知价值 → 点击 CTA → 留资/付费

## 二、核心赚钱路径

```
流量进入 → 上传商品图 → 生成模特图 → 预览价值 → 触发转化 → 留资/付费
```

| 步骤 | 当前状态 | 优化方向 |
|------|----------|----------|
| 1. 用户进入 | 首页可访问 ✅ | 降低跳出率 |
| 2. 上传商品图 | 可上传 ✅ | 引导文案更清晰 |
| 3. 生成模特图 | API 200 ✅ | 成功率 95%+、出图速度 |
| 4. 结果页体验 | 可展示 ✅ | 图片更大、更清晰、加载更快 |
| 5. 触发转化 | CTA 存在 ❓ | 文案更吸引、位置更优 |
| 6. 留资/付费 | 未测试 ❌ | 路径最短化 |

**当前最大瓶颈：步骤 5（CTA 转化）**

## 三、唯一主线

```
稳定出图 → 提升体验 → 触发转化 → 获取收入
```

任何任务必须服务这条链路，否则不执行。

## 四、转化优化机制

| 条件 | 自动触发动作 |
|------|-------------|
| 生成成功但无点击 | Qwen 优化 CTA 文案 |
| 用户停留但未转化 | 优化 result 页面结构/加载速度 |
| 生成失败 | 强制 fallback（占位图+重试按钮） |
| 图片加载慢 | Claude 优化图片压缩/OSS 链路 |

**每轮 Hermes 必须判断：当前最大影响转化的瓶颈是什么**

## 五、Agent 分工

### Claude Code
- 保证 model_photo API 稳定（HTTP 200 + 真实图片返回）
- 修复 diagnosis/execute/result 链路
- 优化图片加载速度（OSS/CDN/压缩）
- 保证 result 页正常展示

### Qwen Code
- 写转化导向的 prompt 模板
- 写 CTA 文案（点击我下载/立即使用等）
- 写 fallback 文案（生成失败时展示）
- 写引导用户拍摄的提示文案
- 优化 result 页结构和信息层级

### Hermes
- 每轮只推进一个转化相关任务
- 判断最大瓶颈（不是做任务列表）
- 控制节奏（禁止并行乱跑）
- 汇总写入 HERMES_REPORT.md

## 六、自动推进规则

1. **无任务时**：自动寻找转化瓶颈
2. **每一轮**：只做一个最关键优化
3. **每个优化必须**：可验证 + 可影响转化

## 七、禁止事项

- 不允许做"看起来很牛但不赚钱"的优化
- 不允许增加复杂度但不提升转化
- 不允许写无用文档（无验收标准）
- 不允许多线程乱跑任务
- 不偏离"出图质量 + 转化"两大目标
- 不空转（无任务时分析数据找瓶颈）

## 八、当前阶段目标

**唯一目标**：把"模特上身图"做成稳定可转化链路

| 验收标准 | 当前状态 |
|----------|----------|
| API 稳定返回图片 | ✅ 已达成 |
| result 页能展示 | ✅ 已达成 |
| 用户看到后有点击行为 | ❌ 未测试 |
| CTA 路径可达 | ❌ 未测试 |

## 九、最终目标（量化）

```
用户上传 → 自动生成 → 感知价值 → 点击 CTA → 留资/付费
```

| 指标 | 目标值 |
|------|--------|
| 生成成功率 | > 95% |
| 页面可用率 | 100% |
| CTA 点击率 | 持续优化中 |
| 转化率 | 持续提升 |

系统自动实现：7×24 小时，商家自助，流水线不间断。
